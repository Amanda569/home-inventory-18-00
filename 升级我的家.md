## 3. GitHub 网页部署步骤

1. 打开 [GitHub](https://github.com)，登录。

2. 右上角 `+` -> `New repository`。

3. 仓库名可以写：

   ```
   home-inventory
   ```

4. 选择 `Public`。

5. 点 `Create repository`。

6. 把 `home-inventory-pwa-deploy.zip` 解压。

7. 在 GitHub 仓库页面点 `uploading an existing file`。

   ![image-20260722143407002](升级我的家.assets/image-20260722143407002.png)

8. 上传解压后的所有文件，不要只上传 zip。【E:\Life\我的家\home-inventory-pwa-deploy-第二版】

9. 点 `Commit changes`。

10. 进入仓库 `Settings` -> `Pages`。

11. `Build and deployment`  里选择：

    - Source: `Deploy from a branch`
    - Branch: `main`
    - Folder: `/ root`

12. 点 `Save`。

13. 等 1-3 分钟，GitHub 会给你一个网址，通常是：

```
# https://你的用户名.github.io/home-inventory/
https://amanda569.github.io/home-inventory/
```

![image-20260722143814290](升级我的家.assets/image-20260722143814290.png)

GitHub 官方文档也说明 Pages 可以从仓库分支发布静态站点：[GitHub Pages publishing source](https://docs.github.com/en/pages/getting-started-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site)。

上传成功后，用手机打开 GitHub Pages 给你的 `https://...` 地址，新增一个测试物品。然后去 Supabase 表里看有没有写入。如果能看到，就说明手机端云保存通了。

## 4. 第二版升级，把数据库的JSON展开成表格

> 上一版在手机更新后，数据库中的数据是在payload里以JSON形式保存，不便于浏览
>
> ![image-20260722144555853](升级我的家.assets/image-20260722144555853.png)

在 Supabase 里这样做：

1. 左侧点 `SQL Editor`
2. 新建 Query
3. 把 [supabase-views.sql](/E:/Life/我的家/supabase-views.sql) 内容复制进去
4. 点 `Run`
5. 回到左侧 `Table Editor`

之后会多出 3 个可浏览的视图：

```
inventory_items_view
inventory_replenish_view
inventory_locations_view
```

分别是：

- `inventory_items_view`：一行一个物品，能直接看名称、分类、位置、数量、是否需补货
- `inventory_replenish_view`：只看需要补货的物品
- `inventory_locations_view`：按位置汇总，适合看“床下-A箱里有什么”

这些是只读看板，日常新增、修改、删除还是在你的 PWA 页面里做。

```

```

已经整理好了。

现在 `E:\Life\我的家` 是一个真正的本地 Git 仓库了，`origin` 已经指向你的仓库：
`https://github.com/Amanda569/home-inventory-18-00.git`

我还帮你加了一个一键脚本：
[sync-to-github.ps1](E:/Life/我的家/sync-to-github.ps1)

以后你在这个目录里直接跑：【PowerShell】

```
.\sync-to-github.ps1 "你的更新说明"
```

它会自动执行 `git add`、`git commit`、`git push`。
